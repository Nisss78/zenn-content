## はじめに

AIエージェントを使っていると、「外部のAPIやデータベース、社内ツールにもアクセスできたらもっと便利なのに」と思ったことはありませんか？

**MCP（Model Context Protocol）** は、AIモデルと外部ツールを標準的な方法で接続するためのオープンプロトコルです。OpenClawはMCPに対応しており、`mcporter` スキルを使うことで、様々な外部サーバーやツールをAIエージェントから呼び出せるようになります。

この記事では、**OpenClawでMCPサーバーを利用する方法**を、初心者向けにステップバイステップで解説します。コード例を中心に、実際に手を動かして学べる構成です。

---

## MCPとは何か？ — 30秒で理解する

MCPは、Anthropic社が提唱した「AIモデルと外部ツールをつなぐ共通規格」です。

### 従来の課題

```
AIモデル → 各ツールごとに専用の統合コードが必要
            （非効率、保守が大変）
```

### MCPの解決策

```
AIモデル → MCPクライアント → MCPサーバー → 外部ツール
                ↑ 標準プロトコル ↑
```

一度MCPサーバーを用意すれば、**MCP対応のどのAIツールからでも利用可能**になります。

## ステップ1：mcporterスキルの確認

OpenClawにはMCPを扱うための `mcporter` スキルが標準で用意されています。まずはスキルが利用可能か確認しましょう。

### スキルの読み込み

OpenClawのチャットで以下のように依頼します：

```
MCPサーバーを一覧表示して
```

または、SKILL.mdを直接確認します：

```bash
# スキルの場所を確認
ls ~/.openclaw/workspace/skills/mcporter/
```

出力例：

```
SKILL.md
scripts/
config/
```

### mcporterの基本コマンド

```bash
# 利用可能なMCPサーバーを一覧表示
mcporter list

# 特定のサーバーの詳細を表示
mcporter info <server-name>

# サーバーを呼び出してツールを実行
mcporter call <server-name> <tool-name> --arg '{"key": "value"}'
```

## ステップ2：最初のMCPサーバーを追加する

一番簡単な例として、**ファイルシステムMCPサーバー**を追加してみましょう。これにより、AIエージェントがローカルのファイルを読み書きできるようになります。

### サーバーの追加

```bash
# ファイルシステムサーバーを追加
mcporter add filesystem \
  --command "npx" \
  --args "-y @modelcontextprotocol/server-filesystem /home/user/documents"
```

これだけで、OpenClawのAIエージェントが `/home/user/documents` 以下のファイルにアクセスできるようになります。

### 設定ファイルの確認

MCPサーバーの設定は設定ファイルに保存されます：

```bash
cat ~/.openclaw/mcp-config.json
```

内容例：

```json
{
  "servers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/documents"],
      "enabled": true
    }
  }
}
```

> ⚠️ セキュリティ上の注意：ファイルシステムサーバーは、指定したディレクトリのみアクセス可能です。ホームディレクトリ全体を指定すると、機密ファイルにアクセスされるリスクがあるため注意しましょう。

## ステップ3：Web検索サーバーを追加する

次に、**Brave Search MCPサーバー**を追加して、AIエージェントにWeb検索能力を付与しましょう。

### API キーの取得

1. [Brave Search API](https://brave.com/search/api/) にアクセス
2. 無料プラン（月2,000クエリ）でサインアップ
3. APIキーを取得

### サーバーの追加

```bash
mcporter add brave-search \
  --command "npx" \
  --args "-y @modelcontextprotocol/server-brave-search" \
  --env '{"BRAVE_API_KEY": "your-api-key-here"}'
```

### 動作確認

OpenClawのチャットで：

```
Brave Searchで「OpenClaw MCP」について検索して
```

AIエージェントがMCP経由でBrave Searchを呼び出し、結果を返してくれます。

## ステップ4：GitHub MCPサーバーでリポジトリ操作

開発者にとって特に便利なのが、**GitHub MCPサーバー**です。これを追加すると、AIエージェントから直接Issueの作成、PRのレビュー、ファイルの操作が可能になります。

### サーバーの追加

```bash
mcporter add github \
  --command "npx" \
  --args "-y @modelcontextprotocol/server-github" \
  --env '{"GITHUB_PERSONAL_ACCESS_TOKEN": "ghp_your_token_here"}'
```

### トークンに必要な権限

GitHubのPersonal Access Tokenには以下の権限が必要です：

| 権限 | 用途 |
|------|------|
| `repo` | リポジトリへのフルアクセス |
| `read:org` | 組織情報の読み取り |
| `workflow` | GitHub Actionsの操作 |

### 使い方の例

OpenClawのチャットで以下のように依頼できます：

```
githubサーバーで、openclawリポジトリの最新のIssueを5件表示して
```

```
githubサーバーで、mainブランチの最新コミットメッセージを確認して
```

## ステップ5：複数サーバーを組み合わせた実践例

ここまでで3つのMCPサーバーを追加しました。これらを組み合わせることで、より強力なワークフローが構築できます。

### 例：リサーチ＆ドキュメント作成ワークフロー

```
以下のタスクを実行して：
1. brave-searchで「2026年 AIエージェント フレームワーク 比較」を検索
2. 検索結果をまとめてMarkdown形式で整理
3. filesystemサーバーで /home/user/documents/research.md に保存
4. githubサーバーでリポジトリにコミット＆プッシュ
```

OpenClawは各ステップで適切なMCPサーバーを自動的に選択し、連携して実行します。

### 通信の流れ

```
ユーザーの指示
    ↓
OpenClaw（AIエージェント）
    ├── brave-search MCP → Web検索結果を取得
    ├── filesystem MCP → ローカルファイルに保存
    └── github MCP → リポジトリにプッシュ
```

## ステップ6：独自のMCPサーバーを作る

既存のサーバーだけでなく、独自のMCPサーバーも簡単に作れます。例えば、社内APIをMCPサーバーとして公開するケースを見てみましょう。

### 最小構成のMCPサーバー（TypeScript）

```typescript
import { Server } from "@modelcontextprotocol/sdk/server/index.js";
import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";

const server = new Server({
  name: "my-custom-server",
  version: "1.0.0",
});

// ツールの定義
server.setRequestHandler("tools/list", async () => ({
  tools: [
    {
      name: "get_weather",
      description: "指定した都市の天気を取得する",
      inputSchema: {
        type: "object",
        properties: {
          city: { type: "string", description: "都市名" },
        },
        required: ["city"],
      },
    },
  ],
}));

// ツールの実行
server.setRequestHandler("tools/call", async (request) => {
  if (request.params.name === "get_weather") {
    const city = request.params.arguments.city;
    // ここに実際のAPI呼び出しを実装
    return {
      content: [
        {
          type: "text",
          text: `${city}の天気：晴れ、気温22°C`,
        },
      ],
    };
  }
});

// サーバー起動
const transport = new StdioServerTransport();
await server.connect(transport);
```

### 独自サーバーの登録

```bash
# ビルド後
mcporter add my-weather \
  --command "node" \
  --args "/path/to/my-weather-server/dist/index.js"
```

## トラブルシューティング

よくある問題と解決策をまとめました。

### サーバーが認識されない

```bash
# 設定ファイルの確認
mcporter list

# 設定ファイルの場所
ls ~/.openclaw/mcp-config.json
```

**原因**: `enabled: false` になっている可能性があります。設定ファイルで `true` に変更してください。

### 接続エラーが出る

```
Error: MCP server connection failed
```

**対処法**:

1. npxコマンドが実行できるか確認：`npx --version`
2. ネットワーク接続を確認
3. APIキーが正しいか確認
4. サーバーのパッケージが最新か確認：`npx -y @modelcontextprotocol/server-filesystem@latest`

### 環境変数が反映されない

`--env` で渡した環境変数が反映されない場合、設定ファイルを直接編集します：

```json
{
  "servers": {
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-brave-search"],
      "env": {
        "BRAVE_API_KEY": "BSA_xxxxxxxx"
      },
      "enabled": true
    }
  }
}
```

## ベストプラクティス

### セキュリティ

- **最小権限の原則**: ファイルシステムサーバーは必要なディレクトリのみ指定
- **APIキーの管理**: 環境変数で渡し、ソースコードにハードコードしない
- **本番環境では**: 認証・認可を適切に設定した独自サーバーを推奨

### パフォーマンス

- **サーバーを使わない時は無効化**: `enabled: false` でリソース節約
- **キャッシュの活用**: 同じ検索結果を何度も取得しないよう、エージェントに指示
- **並列呼び出し**: 複数の独立したツール呼び出しは並列で実行可能

### 運用

- **設定ファイルのバージョン管理**: Gitで管理し、チームで共有
- **ログの監視**: MCPサーバーの呼び出しログを定期的に確認
- **定期的なアップデート**: MCPサーバーパッケージの最新版を追従
