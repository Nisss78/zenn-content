# OpenClawでZenn記事を書いてXに自動投稿する完全ガイド

## はじめに

AIアシスタント「OpenClaw」とZenn CLI、X（Twitter）APIを組み合わせることで、**記事の執筆から投稿、SNSでの告知まで完全自動化**できます。

この記事では、以下のフローを実現する方法を解説します：

```
OpenClaw（AI執筆）→ Zenn（技術ブログ）→ X（告知投稿）
```

### なぜこの構成なのか

- **OpenClaw**: Markdownで記事を生成・管理
- **Zenn CLI**: GitHub連携で自動デプロイ
- **X API**: 投稿完了を自動告知

開発者にとって、技術記事の執筆〜発信までの工数を大幅に削減できます。

---

## 前提条件

以下の環境が必要です：

### 必須

- **Node.js** v18以上
- **Git**
- **GitHubアカウント**
- **Zennアカウント**（GitHub連携済み）
- **OpenClaw**（ローカルインストール済み）

### オプション（X自動投稿用）

- **X Developer Account**（Basic以上推奨）
- API Key / API Secret / Access Token / Access Token Secret

---

## Step 1: Zenn投稿環境のセットアップ

### 1-1. zenn-cliのインストール

```bash
npm install -g zenn-cli
```

### 1-2. リポジトリの初期化

```bash
mkdir zenn-content && cd zenn-content
zenn init
```

以下の構造が作成されます：

```
zenn-content/
├── articles/    # 記事用ディレクトリ
├── books/       # 本用ディレクトリ
└── README.md
```

### 1-3. GitHub連携

1. GitHubでリポジトリを作成（例：`username/zenn-content`）
2. Pushする

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/zenn-content.git
git push -u origin main
```

3. ZennのダッシュボードでGitHub連携を有効化
   - https://zenn.dev/dashboard/setup

---

## Step 2: OpenClaw × Zenn 連携

### 2-1. 記事の作成

OpenClawに以下のように依頼します：

```
Zennに「PythonでCLIツールを作る方法」という記事を投稿して
```

OpenClawが自動で：

1. `zenn new:article` を実行
2. Markdownを生成
3. Git commit & push

### 2-2. 記事フォーマット

```markdown
---
title: "記事タイトル"
emoji: "🚀"
type: "tech"  # tech または idea
topics: ["python", "cli"]
published: true  # falseで下書き
---

# 本文

Markdownで書く...
```

### 2-3. 公開のポイント

**重要**: `published: true` にしないとリンクが生成されません。

```yaml
published: true  # ← これが必要！
```

---

## Step 3: X自動投稿の実装

### 3-1. X APIの準備

1. [X Developer Portal](https://developer.twitter.com/)でアプリを作成
2. API Key & Secret を取得
3. Access Token & Secret を生成

### 3-2. Python + tweepy で投稿

```python
import tweepy

# 認証
client = tweepy.Client(
    bearer_token="YOUR_BEARER_TOKEN",
    consumer_key="YOUR_API_KEY",
    consumer_secret="YOUR_API_SECRET",
    access_token="YOUR_ACCESS_TOKEN",
    access_token_secret="YOUR_ACCESS_SECRET"
)

# 投稿
article_url = "https://zenn.dev/username/articles/xxxxx"
tweet = f"""
📝 新着記事

{article_url}

#OpenClaw #Zenn #技術ブログ
"""

client.create_tweet(text=tweet)
```

### 3-3. OpenClawスキルとして統合

OpenClawのスキル機能を使うと、Zenn投稿後に自動でXへ投稿できます：

```
OpenClawに「Zenn記事投稿してXにも告知して」と依頼
```

内部フロー：
1. Zenn CLIで記事作成
2. `published: true` でcommit & push
3. 記事URLを取得
4. X APIでツイート

## トラブルシューティング

### Q. Zennに反映されない

A. 以下を確認：

- GitHubにpushされているか
- Zennダッシュボードで連携が有効か
- `published: true` になっているか

### Q. 記事URLが取得できない

A. `published: false`（下書き）の可能性があります。公開状態に変更してpushし直してください。

### Q. X APIでエラー

A. Freeプランでは書き込みできません。Basic以上への課金が必要です。

## まとめ

OpenClaw × Zenn CLI × X API の組み合わせで、技術記事の執筆〜発信まで自動化できます。

**構築のポイント：**

1. Zenn CLI + GitHub連携をセットアップ
2. `published: true` で公開投稿
3. X API（Basic以上）で告知自動化

これで「記事書いたけど告知忘れた」を防げます。
