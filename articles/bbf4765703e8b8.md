# 【OpenClaw実践チュートリアル】ファイル操作とAPI連携のベストプラクティス

こんにちは、OpenClawを使い始めたばかりの開発者さんへ。

この記事では、OpenClawの核心機能である**ファイル操作**と**API連携**について、実際のコード例と共に解説します。初心者の方でも理解できるように、基本概念から実践的なベストプラクティスまで段階的に学んでいきましょう。

## はじめに：なぜOpenClawなのか

OpenClawは、単なるAIアシスタントではありません。**自動化された開発環境**として、ファイル操作、API連携、プロジェクト管理といった開発の根幹をなすタスクを効率化する強力なツールです。

多くの開発者が「ファイル操作は面倒」「API連携は複雑」と感じている問題を、OpenClawは以下のように解決します：

- **自動化されたファイル管理**：手動での操作を最小化
- **スマートなAPI連携**：複雑な認証処理を自動化
- **バージョン管理との連携**：Git操作とのシームレスな統合
- **エラーハンドリングの標準化**：信頼性の高いコードを生成

## 基本概念：OpenClawのアーキテクチャ

### 1. ワークスペースの概念

OpenClawは**ワークスペースベース**で動作します。現在の作業ディレクトリがすべての操作の中心となります。

```
/home/unosh/.openclaw/workspace/
├── project1/          # プロジェクト1
├── project2/          # プロジェクト2
└── temp/              # 一時作業領域
```

### 2. ファイル操作の基本パターン

OpenClawのファイル操作は3つの基本パターンに分けられます：

1. **Read**: 既存ファイルの読み込み
2. **Write**: 新規ファイルの作成・上書き
3. **Edit**: 既存ファイルの編集

## 第1章：ファイル操作のベストプラクティス

### 1.1 基本的なファイル操作

#### ファイルの読み込み

最も基本的な操作はファイルの読み込みです。OpenClawでは`read`コマンドを使用します。

```python
# ファイルを読み込む基本的なパターン
def read_file_safely(file_path):
    """
    ファイルを安全に読み込む関数
    
    Args:
        file_path (str): 読み込むファイルのパス
        
    Returns:
        str: ファイル内容、読み込み失敗時はNone
    """
    try:
        content = read(file_path=file_path)
        return content
    except FileNotFoundError:
        print(f"エラー: ファイル {file_path} が見つかりません")
        return None
    except Exception as e:
        print(f"エラー: ファイル読み込みに失敗しました - {e}")
        return None

# 使用例
content = read_file_safely("data/config.json")
if content:
    print("ファイルの内容:")
    print(content)
```

```python
def batch_process_directory(directory_path, process_function):
    """
    指定ディレクトリ内の全ファイルを処理する
```

#### バックアップとリストア

重要なファイルのバックアップを作成・復元するパターンです。

```python
import shutil
from datetime import datetime

def backup_file(file_path, backup_dir="backups"):
    """
    ファイルをバックアップする
    
    Args:
        file_path (str): バックアップ対象ファイル
        backup_dir (str): バックアップディレクトリ
    """
    try:
        # バックアップディレクトリを作成
        os.makedirs(backup_dir, exist_ok=True)
        
        # タイムスタンプを付けてバックアップファイル名を作成
        timestamp = datetime.now().strftime("%Y%m%d_%H%M%S")
        backup_filename = f"{os.path.basename(file_path)}.backup_{timestamp}"
        backup_path = os.path.join(backup_dir, backup_filename)
        
        # ファイルをコピー
        shutil.copy2(file_path, backup_path)
        
        print(f"バックアップ完了: {file_path} -> {backup_path}")
        return backup_path
        
    except Exception as e:
        print(f"バックアップエラー: {e}")
        return None

def restore_file(backup_path, original_path):
    """
    バックアップファイルから元のファイルを復元する
    
    Args:
        backup_path (str): バックアップファイルのパス
        original_path (str): 復元先のパス
    """
    try:
        # バックアップファイルから読み込み
        backup_content = read(file_path=backup_path)
        
        # 元のファイルに書き戻し
        write(file_path=original_path, content=backup_content)
        
        print(f"復元完了: {backup_path} -> {original_path}")
        return True
        
    except Exception as e:
        print(f"復元エラー: {e}")
        return False
```

## 第2章：API連携のベストプラクティス

### 2.1 基本的なAPI連携

#### HTTPリクエストの実行

OpenClawから外部APIを呼び出す基本的なパターンです。

```python
import json
import requests

def call_api_safely(url, method="GET", headers=None, data=None):
    """
    APIを安全に呼び出す関数
    
    Args:
        url (str): APIエンドポイントURL
        method (str): HTTPメソッド (GET, POST, PUT, DELETE)
        headers (dict): リクエストヘッダー
        data (dict): リクエストボディデータ
        
    Returns:
        dict: APIレスポンス、エラー時はNone
    """
    try:
        # リクエストオプションの準備
        options = {
            'url': url,
            'method': method.upper(),
            'headers': headers or {'Content-Type': 'application/json'}
        }
        
        # POST/PUTの場合はデータを設定
        if data and method.upper() in ['POST', 'PUT']:
            options['data'] = json.dumps(data)
        
        # APIリクエストを実行
        response = requests.request(**options)
        
        # レスポンスのステータスチェック
        response.raise_for_status()
        
        # JSONレスポンスをパース
        return response.json()
        
    except requests.exceptions.RequestException as e:
        print(f"APIリクエストエラー: {e}")
        return None
    except json.JSONDecodeError as e:
        print(f"JSONパースエラー: {e}")
        return None

# 使用例
def get_user_info(user_id):
    """ユーザー情報を取得する"""
    url = f"https://api.example.com/users/{user_id}"
    headers = {
        'Authorization': 'Bearer your-api-token-here',
        'Accept': 'application/json'
    }
    
    response = call_api_safely(url, method="GET", headers=headers)
    return response

# 使用例
user_data = get_user_info("12345")
if user_data:
    print(f"ユーザー名: {user_data.get('name', 'Unknown')}")
```

#### レートリミットの処理

APIのレートリミットを考慮したリクエスト処理です。

```python
import time
from datetime import datetime, timedelta

class APIClient:
    def __init__(self, base_url, api_token):
        self.base_url = base_url
        self.api_token = api_token
        self.last_request_time = None
        self.min_request_interval = 1.0  # 最小リクエスト間隔（秒）
        
    def _wait_if_needed(self):
        """レートリミットに従って待機する"""
        if self.last_request_time:
            elapsed = time.time() - self.last_request_time
            if elapsed < self.min_request_interval:
                wait_time = self.min_request_interval - elapsed
                print(f"レートリミット対応: {wait_time:.1f}秒待機")
                time.sleep(wait_time)
    
    def _make_request(self, endpoint, method="GET", data=None):
        """実際のAPIリクエストを行う"""
        self._wait_if_needed()
        
        url = f"{self.base_url}{endpoint}"
        headers = {
            'Authorization': f'Bearer {self.api_token}',
            'Content-Type': 'application/json'
        }
        
        try:
            response = requests.request(
                method=method,
                url=url,
                headers=headers,
                json=data
            )
            
            self.last_request_time = time.time()
            
            if response.status_code == 429:
                retry_after = int(response.headers.get('Retry-After', 60))
                print(f"レートリミットに達しました: {retry_after}秒後に再試行")
                time.sleep(retry_after)
                return self._make_request(endpoint, method, data)
            
            response.raise_for_status()
            return response.json()
            
        except requests.exceptions.RequestException as e:
            print(f"APIリクエストエラー: {e}")
            return None

# 使用例
api_client = APIClient(
    base_url="https://api.example.com/v1",
    api_token="your-api-token-here"
)

# 複数ユーザー情報を取得
user_ids = ["123", "456", "789"]
for user_id in user_ids:
    user_data = api_client._make_request(f"/users/{user_id}")
    if user_data:
        print(f"ユーザー {user_id}: {user_data.get('name')}")
```

```python
class APIKeyAuthClient:
    def __init__(self, base_url, api_key):
        self.base_url = base_url
        self.api_key = api_key
        
    def _get_headers(self):
        """認証用ヘッダーを取得"""
        return {
            'X-API-Key': self.api_key,
            'Content-Type': 'application/json'
        }
    
    def get_data(self, endpoint, params=None):
        """データをGETで取得"""
```

```python
import requests
from urllib.parse import urlencode

class OAuth2Client:
    def __init__(self, client_id, client_secret, redirect_uri):
        self.client_id = client_id
        self.client_secret = client_secret
        self.redirect_uri = redirect_uri
        self.access_token = None
        
    def get_authorization_url(self, scopes):
        """認証URLを生成"""
        auth_params = {
            'client_id': self.client_id,
            'redirect_uri': self.redirect_uri,
            'response_type': 'code',
            'scope': ' '.join(scopes)
        }
        return f"https://api.example.com/oauth/authorize?{urlencode(auth_params)}"
    
    def exchange_code_for_token(self, authorization_code):
        """認証コードをアクセストークンと交換"""
        token_url = "https://api.example.com/oauth/token"
        
        data = {
            'grant_type': 'authorization_code',
            'code': authorization_code,
            'redirect_uri': self.redirect_uri,
            'client_id': self.client_id,
            'client_secret': self.client_secret
        }
        
        try:
            response = requests.post(token_url, data=data)
            response.raise_for_status()
            token_data = response.json()
            self.access_token = token_data['access_token']
            return self.access_token
        except requests.exceptions.RequestException as e:
            print(f"トークン取得エラー: {e}")
            return None
    
    def make_authenticated_request(self, endpoint, method="GET", data=None):
        """認証済みAPIリクエスト"""
        if not self.access_token:
            raise ValueError("アクセストークンが設定されていません")
        
        url = f"https://api.example.com{endpoint}"
        headers = {
            'Authorization': f'Bearer {self.access_token}',
            'Content-Type': 'application/json'
        }
        
        try:
            response = requests.request(method, url, headers=headers, json=data)
            response.raise_for_status()
            return response.json()
        except requests.exceptions.RequestException as e:
            print(f"APIリクエストエラー: {e}")
            return None
```

## 第3章：エラーハンドリングとデバッグ

### 3.1 例外処理のパターン

ファイル操作とAPI連携における例外処理のベストプラクティスです。

```python
class FileOperationError(Exception):
    """ファイル操作に関するエラー"""
    pass

class APIConnectionError(Exception):
    """API接続に関するエラー"""
    pass

def safe_file_operation(file_path, operation, **kwargs):
    """
    安全なファイル操作を行う
    
    Args:
        file_path (str): ファイルパス
        operation (str): 操作タイプ ('read', 'write', 'edit')
        **kwargs: 操作に必要な追加引数
        
    Returns:
        操作結果、エラー時はNone
        
    Raises:
        FileOperationError: ファイル操作が失敗した場合
    """
    try:
        if operation == 'read':
            content = read(file_path=file_path)
            return content
        elif operation == 'write':
            write(file_path=file_path, content=kwargs.get('content', ''))
            return True
        elif operation == 'edit':
            edit(
                file_path=file_path,
                oldText=kwargs.get('oldText'),
                newText=kwargs.get('newText')
            )
            return True
        else:
            raise FileOperationError(f"不明な操作: {operation}")
            
    except Exception as e:
        raise FileOperationError(f"ファイル操作エラー: {e}")

def safe_api_request(url, method="GET", **kwargs):
    """
    安全なAPIリクエストを行う
    
    Args:
        url (str): API URL
        method (str): HTTPメソッド
        **kwargs: requests.requestに渡す追加引数
        
    Returns:
        レスポンスデータ、エラー時はNone
        
    Raises:
        APIConnectionError: API接続が失敗した場合
    """
    try:
        response = requests.request(method, url, **kwargs)
        response.raise_for_status()
        
        # JSONレスポンスを返す
        try:
            return response.json()
        except ValueError:
            return response.text
            
    except requests.exceptions.HTTPError as e:
        error_msg = f"HTTPエラー: {e.response.status_code}"
        if e.response.status_code == 429:
            error_msg += " (レートリミットに達しました)"
        elif e.response.status_code == 401:
            error_msg += " (認証に失敗しました)"
        elif e.response.status_code == 403:
            error_msg += " (アクセスが拒否されました)"
        raise APIConnectionError(error_msg)
    except requests.exceptions.RequestException as e:
        raise APIConnectionError(f"リクエストエラー: {e}")

# 使用例
try:
    # ファイル操作の例
    content = safe_file_operation("data/config.json", "read")
    print(f"ファイル内容: {content}")
    
    # APIリクエストの例
    api_data = safe_api_request(
        "https://api.example.com/data",
        headers={'Authorization': 'Bearer token'}
    )
    print(f"APIデータ: {api_data}")
    
except FileOperationError as e:
    print(f"ファイル操作失敗: {e}")
except APIConnectionError as e:
    print(f"API接続失敗: {e}")
except Exception as e:
    print(f"予期せぬエラー: {e}")
```

### 3.2 ログ記録のパターン

操作の追跡とデバッグのためのログ記録です。

```python
import logging
from datetime import datetime

class OpenClawLogger:
    def __init__(self, log_file="openclaw_operations.log"):
        self.log_file = log_file
        self.setup_logger()
        
    def setup_logger(self):
        """ロガーを設定"""
        logging.basicConfig(
            level=logging.INFO,
            format='%(asctime)s - %(levelname)s - %(message)s',
            handlers=[
                logging.FileHandler(self.log_file),
                logging.StreamHandler()
            ]
        )
        self.logger = logging.getLogger(__name__)
    
    def log_file_operation(self, operation, file_path, success=True, error_msg=None):
        """ファイル操作をログに記録"""
        status = "SUCCESS" if success else "FAILED"
        message = f"{operation} - {file_path} - {status}"
        
        if not success:
            message += f" - Error: {error_msg}"
            
        self.logger.info(message)
    
    def log_api_request(self, url, method, success=True, error_msg=None, response_time=None):
        """APIリクエストをログに記録"""
        status = "SUCCESS" if success else "FAILED"
        message = f"API {method} - {url} - {status}"
        
        if response_time:
            message += f" ({response_time:.2f}s)"
            
        if not success:
            message += f" - Error: {error_msg}"
            
        self.logger.info(message)

# 使用例
logger = OpenClawLogger()

# ファイル操作のログ記録
try:
    content = read(file_path="config.json")
    logger.log_file_operation("READ", "config.json", success=True)
except Exception as e:
    logger.log_file_operation("READ", "config.json", success=False, error_msg=str(e))

# APIリクエストのログ記録
import time
start_time = time.time()
try:
    response = requests.get("https://api.example.com/data")
    response_time = time.time() - start_time
    logger.log_api_request("https://api.example.com/data", "GET", success=True, response_time=response_time)
except Exception as e:
    response_time = time.time() - start_time
    logger.log_api_request("https://api.example.com/data", "GET", success=False, error_msg=str(e), response_time=response_time)
```

```python
class DataSyncManager:
    def __init__(self, config_file="sync_config.json"):
        self.config = self.load_config(config_file)
        self.logger = OpenClawLogger()
        
    def load_config(self, config_file):
        """設定ファイルを読み込む"""

            content = read(file_path=config_file)
            return json.loads(content)
        except Exception as e:
            self.logger.log_file_operation("READ", config_file, success=False, error_msg=str(e))
            return self.create_default_config()
    
    def create_default_config(self):
        """デフォルト設定を作成"""
        default_config = {
            "api_endpoints": [
                {"name": "users", "url": "https://api.example.com/users"},
                {"name": "products", "url": "https://api.example.com/products"}
            ],
            "sync_interval": 3600,  # 1時間
            "data_dir": "sync_data"
        }
        write(file_path="sync_config.json", content=json.dumps(default_config, indent=2))
        return default_config
    
    def sync_all_data(self):
        """全データを同期する"""
        for endpoint in self.config["api_endpoints"]:
            try:
                self.sync_endpoint_data(endpoint)
            except Exception as e:
                self.logger.log_api_request(endpoint["url"], "SYNC", success=False, error_msg=str(e))
    
    def sync_endpoint_data(self, endpoint):
        """特定のエンドポイントのデータを同期する"""
        url = endpoint["name"]
        api_url = endpoint["url"]
        
        try:
            # APIからデータを取得
            data = safe_api_request(api_url, headers={"Authorization": f"Bearer {self.config['api_token']}"})
            
            if data:
                # ローカルファイルに保存
                local_file = f"{self.config['data_dir']}/{url}.json"
                write(file_path=local_file, content=json.dumps(data, indent=2))
                
                self.logger.log_api_request(api_url, "SYNC", success=True)
                print(f"同期完了: {url}")
            else:
                raise Exception("APIからデータを取得できませんでした")
                
        except Exception as e:
            self.logger.log_api_request(api_url, "SYNC", success=False, error_msg=str(e))
            raise
```

## 第5章：パフォーマンス最適化

### 5.1 キャッシュの実装

APIリクエストの結果をキャッシュしてパフォーマンスを向上させるパターンです。

```python
import hashlib
import json
from pathlib import Path

class APICache:
    def __init__(self, cache_dir="cache", max_age=3600):
        self.cache_dir = Path(cache_dir)
        self.max_age = max_age  # 最大保存時間（秒）
        self.cache_dir.mkdir(exist_ok=True)
        
    def _get_cache_key(self, url, params=None):
        """キャッシュキーを生成"""
        cache_data = f"{url}_{json.dumps(params or {}, sort_keys=True)}"
        return hashlib.md5(cache_data.encode()).hexdigest()
    
    def _get_cache_path(self, cache_key):
        """キャッシュファイルパスを取得"""
        return self.cache_dir / f"{cache_key}.json"
    
    def get_from_cache(self, url, params=None):
        """キャッシュからデータを取得"""
        cache_key = self._get_cache_key(url, params)
        cache_path = self._get_cache_path(cache_key)
        
        if not cache_path.exists():
            return None
            
        try:
            cache_content = read(file_path=str(cache_path))
            cache_data = json.loads(cache_content)
            
            # キャッシュの有効期限をチェック
            cache_time = cache_data.get('timestamp', 0)
            if time.time() - cache_time > self.max_age:
                return None
                
            return cache_data.get('data')
            
        except Exception:
            return None
    
    def save_to_cache(self, url, data, params=None):
        """データをキャッシュに保存"""
        cache_key = self._get_cache_key(url, params)
        cache_path = self._get_cache_path(cache_key)
        
        cache_data = {
            'timestamp': time.time(),
            'url': url,
            'params': params,
            'data': data
        }
        
        write(file_path=str(cache_path), content=json.dumps(cache_data, indent=2))
    
    def make_cached_request(self, url, method="GET", params=None, **kwargs):
        """キャッシュ付きAPIリクエスト"""
        # まずキャッシュをチェック
        cached_data = self.get_from_cache(url, params)
        if cached_data:
            print(f"キャッシュから読み込み: {url}")
            return cached_data
        
        # キャッシュにない場合はAPIリクエスト
        data = safe_api_request(url, method=method, params=params, **kwargs)
        
        if data:
            # 成功したらキャッシュに保存
            self.save_to_cache(url, data, params)
            
        return data

# 使用例
cache = APICache(max_age=1800)  # 30分間有効

# キャッシュ付きAPIリクエスト
users = cache.make_cached_request(
    "https://api.example.com/users",
    headers={"Authorization": "Bearer token"}
)
```

### 5.2 並列処理の実装

複数のAPIリクエストを並列で実行してパフォーマンスを向上させるパターンです。

```python
import concurrent.futures
import threading

class ParallelAPIClient:
    def __init__(self, max_workers=5):
        self.max_workers = max_workers
        self.logger = OpenClawLogger()
        self.lock = threading.Lock()
        
    def fetch_multiple_urls(self, urls, headers=None):
        """複数URLを並列で取得"""
        results = {}
        
        def fetch_single_url(url):
            try:
                response = safe_api_request(url, headers=headers or {})
                
                # スレッドセーフに結果を保存
                with self.lock:
                    results[url] = response
                    
                self.logger.log_api_request(url, "GET", success=True)
                return response
                
            except Exception as e:
                with self.lock:
                    results[url] = None
                    
                self.logger.log_api_request(url, "GET", success=False, error_msg=str(e))
                return None
        
        # スレッドプールを使用して並列処理
        with concurrent.futures.ThreadPoolExecutor(max_workers=self.max_workers) as executor:
            # 全タスクを登録
            future_to_url = {
                executor.submit(fetch_single_url, url): url 
                for url in urls
            }
            
            # 完了を待機
            for future in concurrent.futures.as_completed(future_to_url):
                url = future_to_url[future]
                try:
                    result = future.result()
                except Exception as e:
                    self.logger.log_api_request(url, "GET", success=False, error_msg=str(e))
                    
        return results

# 使用例
parallel_client = ParallelAPIClient(max_workers=3)

# 複数のAPIエンドポイントを並列で取得
urls = [
    "https://api.example.com/users/1",
    "https://api.example.com/users/2", 
    "https://api.example.com/users/3",
    "https://api.example.com/products",
    "https://api.example.com/categories"
]

results = parallel_client.fetch_multiple_urls(urls)
print(f"取得結果: {len([r for r in results.values() if r])}件成功")
```
