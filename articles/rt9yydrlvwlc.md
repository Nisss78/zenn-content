## 基本コマンド

### `/help` - ヘルプを表示

利用可能なコマンドとオプションの一覧を表示します。

```
/help
```

**活用シーン:**
- コマンドを忘れた時
- 新しいバージョンで追加されたコマンドを確認したい時

---

### `/clear` - 会話をクリア

現在の会話履歴を削除し、新しいセッションを開始します。

```
/clear
```

**注意点:**
- クリアすると会話履歴は復元できません

---

### `/compact` - 会話を要約

長い会話履歴を要約し、コンテキストを圧縮します。

```
/compact
```

**活用シーン:**
- 長時間のセッションでコンテキストが肥大化した時
- コストを抑えつつ会話を継続したい時

---

### `/cost` - セッションのコストを表示

現在のセッションで使用したトークン数と推定コストを表示します。

```
/cost
```

## セッション管理

### `/status` - ステータスを表示

現在のセッション状態、モデル情報、設定などを表示します。

```
/status
```

---

### `/init` - プロジェクトを初期化

カレントディレクトリにClaude Code用の設定ファイル（CLAUDE.md）を作成します。

```
/init
```

**CLAUDE.mdの例:**
```markdown
# プロジェクト: My App

## 技術スタック
- React 18
- TypeScript
- Tailwind CSS

## コーディング規約
- 関数コンポーネントを使用
```

## モデル・パフォーマンス

### `/model` - モデル切り替え

使用するClaudeモデルを切り替えます。

```
/model
```

**各モデルの特徴:**

| モデル | 特徴 | 向いているタスク |
|--------|------|-----------------|
| **Sonnet** | バランス型 | 通常の開発、コード生成 |
| **Opus** | 最高性能 | 複雑な設計、難しい問題 |
| **Haiku** | 高速・低コスト | 単純なタスク、素早い確認 |

```
# 複雑なリファクタリングはOpusで
/model opus

# 単純な修正はHaikuで
/model haiku
```

---

### `/fast` - 高速モード

同じモデルでより高速な応答を実現します。

```
/fast
```

## プランニング

### `/plan` - プランニングモード

実装前に詳細な計画を立てるモード。

```
/plan
```

**使用例:**
```
User: /plan
> 認証システムを実装して

Claude: ## 実装プラン

### フェーズ1: 基盤構築
1. Userモデルの作成
2. パスワードハッシュ化

### フェーズ2: エンドポイント
1. POST /auth/register
2. POST /auth/login

この計画で進めてよろしいですか？
```
