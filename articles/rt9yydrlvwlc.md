この記事では、Claude Codeの**全標準スラッシュコマンド**を網羅的に解説し、実践的な使用例を紹介します。

## 基本コマンド

### `/help` - ヘルプを表示

**概要:** 利用可能なコマンドとオプションの一覧を表示します。

```
/help
```

**使用例:**
```
User: /help
Claude: 利用可能なコマンド:
  /clear     - 会話をクリア
  /compact   - 会話を要約
  /cost      - セッションのコストを表示
  ...
```

**活用シーン:**
- コマンドを忘れた時
- 新しいバージョンで追加されたコマンドを確認したい時

---

### `/clear` - 会話をクリア

**概要:** 現在の会話履歴を削除し、新しいセッションを開始します。

```
/clear
```

**使用例:**
```
User: /clear
Claude: 会話をクリアしました。新しいセッションを開始します。
```

**活用シーン:**
- 異なるプロジェクトに切り替える時
- コンテキストをリセットして新鮮な状態で始めたい時
- 会話が長すぎて応答が遅くなった時

**注意点:**
- クリアすると会話履歴は復元できません
- 重要な情報は事前にコピーしておきましょう

---

### `/compact` - 会話を要約

**概要:** 長い会話履歴を要約し、コンテキストを圧縮します。重要な情報は保持しつつ、トークン使用量を削減できます。

```
/compact
```

**使用例:**
```
User: /compact
Claude: 会話を要約しました：
- プロジェクト: React Todoアプリ
- 完了したタスク: コンポーネント作成、スタイリング
- 保留中のタスク: テスト作成、デプロイ
```

**活用シーン:**
- 長時間のセッションでコンテキストが肥大化した時
- コストを抑えつつ会話を継続したい時
- 重要な決定事項を整理したい時

---

### `/cost` - セッションのコストを表示

**概要:** 現在のセッションで使用したトークン数と推定コストを表示します。

```
/cost
```

**使用例:**
```
User: /cost
Claude: セッション統計:
- 入力トークン: 45,230
- 出力トークン: 12,450
- 合計トークン: 57,680
- 推定コスト: $0.87
```

**活用シーン:**
- コスト管理
- 長いセッションでのリソース確認
- API使用量のモニタリング

## セッション管理

### `/status` - ステータスを表示

**概要:** 現在のセッション状態、モデル情報、設定などを表示します。

```
/status
```

**使用例:**
```
User: /status
Claude: セッションステータス:
- モデル: claude-sonnet-4-20250514
- ワーキングディレクトリ: /home/user/my-project
- ファイルコンテキスト: 3 files loaded
- 現在のブランチ: feature/auth
```

**活用シーン:**
- 現在の環境を確認したい時
- デバッグ時にどのモデルが使われているか確認
- プロジェクトの状態把握

---

### `/init` - プロジェクトを初期化

**概要:** カレントディレクトリにClaude Code用の設定ファイル（CLAUDE.md）を作成します。

```
/init
```

**使用例:**
```
User: /init
Claude: プロジェクトを初期化しました。
作成されたファイル:
- CLAUDE.md（プロジェクト固有の指示）
- .claude/（設定ディレクトリ）
```

**活用シーン:**
- 新しいプロジェクトを始める時
- 既存プロジェクトにClaude Codeを導入する時
- チームで共有する設定を作成する時

**CLAUDE.mdの例:**
```markdown
# プロジェクト: My App

## 技術スタック
- React 18
- TypeScript
- Tailwind CSS

## コーディング規約
- 関数コンポーネントを使用
- early returnパターンを推奨
```

## コードレビュー・分析

### `/review` - コードレビュー

**概要:** 現在の変更やコードをレビューし、改善点を提案します。

```
/review
```

**使用例:**
```
User: /review
Claude: コードレビュー結果:

## 変更されたファイル
- src/components/AuthForm.tsx

## 改善提案

1. セキュリティ: パスワードフィールドにautocomplete属性を追加
   ```tsx
   <input type="password" autoComplete="new-password" />
   ```

2. パフォーマンス: フォームバリデーションにデバウンスを追加推奨

3. 型安全性: APIレスポンスにZodスキーマの使用を検討
```

**活用シーン:**
- コミット前のセルフレビュー
- PR作成前の品質チェック
- セキュリティ脆弱性の発見

---

### `/pr-comments` - PRコメントを確認

**概要:** 現在のブランチに関連するPRのコメントを取得・表示します。

```
/pr-comments
```

**使用例:**
```
User: /pr-comments
Claude: PR #42 のコメント:

## レビュアー: @senior-dev

> src/api/users.ts:15-20
> エラーハンドリングが不足しています。
> catch ブロックでエラーをログに出力してください。

## 対応状況
- [ ] 未対応: 1件
```

**活用シーン:**
- PRのフィードバック確認
- レビュー指摘への対応
- チーム開発でのコミュニケーション

## システム・環境

### `/doctor` - 環境診断

**概要:** Claude Codeの動作環境を診断し、問題がないか確認します。

```
/doctor
```

**使用例:**
```
User: /doctor
Claude: 環境診断結果:

Claude Code バージョン: 1.0.0
Node.js: v20.10.0
API接続: 正常
Git: 2.43.0
設定ファイル: .claude/config.json が見つかりません

推奨アクション:
- `/init` を実行して設定ファイルを作成してください
```

**活用シーン:**
- インストール後の動作確認
- エラー発生時のトラブルシューティング
- 環境の健全性チェック

---

### `/terminal-setup` - ターミナル設定

**概要:** ターミナルの統合設定を行います。キーバインドやシェル統合などを設定できます。

```
/terminal-setup
```

**使用例:**
```
User: /terminal-setup
Claude: ターミナル設定メニュー:

1. キーバインド設定
2. シェル統合（bash/zsh/fish）
3. カラースキーム
4. 補完設定

選択してください [1-4]:
```

**活用シーン:**
- 初回セットアップ
- キーバインドのカスタマイズ
- シェル統合の有効化

---

### `/vim` - Vimモード切り替え

**概要:** Vimスタイルのキーバインドを有効/無効にします。

```
/vim
```

**使用例:**
```

Claude: Vimモードを有効にしました。
- Esc: ノーマルモードへ
- i: 挿入モードへ
- :w: 保存
```

**活用シーン:**
- Vimユーザー向けの設定
- キーボード操作の効率化

**概要:** ファイルアクセスやコマンド実行の権限を管理します。

```
/permissions
```

**使用例:**
```

- ファイル読み込み: プロジェクトディレクトリのみ

設定を変更しますか？ [y/N]:
```

**活用シーン:**
- セキュリティレベルの調整
- 特定ディレクトリへのアクセス許可
- コマンド実行ポリシーの変更

## 実践的なワークフロー例

### 新規プロジェクト開始時

```bash
# 1. プロジェクトディレクトリへ移動
cd my-new-project

# 2. Claude Code起動（自動的にプロジェクトを読み込む）
claude

# 3. プロジェクト初期化
/init

# 4. 環境確認
/doctor

# 5. 開発開始
# Reactアプリを作成して
```

### PR作成前のチェック

```bash
# 1. 現在の変更を確認
git status

# 2. Claude Codeでレビュー
/review

# 3. セキュリティチェック
> このコードにセキュリティ上の問題がないか確認して

# 4. テスト実行
> テストを実行して

# 5. コミット作成
> これらの変更をコミットして
```

### バグ修正ワークフロー

```bash
# 1. エラーログを共有
> [エラーログを貼り付け]

# 2. 原因分析
> このエラーの原因を特定して

# 3. 修正案提示
> 修正方法を提案して

# 4. 修正実装
> その修正を適用して

# 5. テスト
> 修正が正しいかテストして
```

## Tips & ベストプラクティス

### 1. CLAUDE.mdを活用する

プロジェクト固有の指示をCLAUDE.mdに記載することで、毎回同じ説明をする必要がなくなります。

```markdown
# CLAUDE.md

## プロジェクト概要
ECサイトのフロントエンド開発

## 技術スタック
- Next.js 14 (App Router)
- TypeScript
- Prisma
- Tailwind CSS

## コーディング規約
- 関数はアロー関数で記述
- コンポーネントはPascalCase
- ファイル名はkebab-case

## 禁止事項
- any型の使用
- console.logの残置
```

### 2. 段階的に指示する

複雑なタスクは段階的に分解すると良い結果が得られます。

```
悪い例:
> ECサイトを全部作って

良い例:
> 1. まずプロジェクト構造を設計して
> 2. 次にデータベーススキーマを定義して
> 3. その後APIエンドポイントを作成して
> 4. 最後にUIコンポーネントを実装して
```

### 3. ファイルを明示的に指定する

特定のファイルを対象にする場合は明示的に指定します。

```
> src/components/Header.tsx のスタイルを修正して
```

### 4. コンテキストを適切に管理する

```
# 定期的に要約
/compact

# 不要な会話はクリア
/clear

# コストを意識
/cost
```

### 5. Gitと連携する

Claude CodeはGit操作も得意です。

```
> 変更をステージングして、コミットして
> feature/authブランチを作成して切り替えて
> mainブランチにマージして
```

### 6. エラーは詳細に共有する

エラーが発生した場合は、エラーメッセージ全体を共有しましょう。

```
良い例:
> npm run devを実行したら以下のエラーが出ました：
> 
> Error: Cannot find module '@/components/Header'
> Require stack:
> - /app/page.tsx
> 
> tsconfig.jsonのpaths設定は以下の通りです：
> [設定内容を貼り付け]
```

## まとめ

Claude Codeは単なるチャットボットではなく、本格的な開発ツールです。スラッシュコマンドを活用することで、より効率的に開発を進められます。

**覚えておくべき重要コマンド:**
- `/init` - プロジェクト開始時に必須
- `/review` - コミット前の品質チェックに
- `/compact` - 長いセッションでコンテキスト管理
- `/cost` - コスト意識を持つために

公式ドキュメント: https://docs.anthropic.com/claude-code

Happy Coding with Claude!
